Classes and Modules

A brief comparison

A module shares many of the features of a full-fledged class, but it is actually more like half-fledged. That is, it contains constants, variables, and methods, but it cannot be instantiated on its own. In other words, this results in a NoMethodError:

  • module Mod
  • def say_hello
  • puts "Hello"
  • end
  • end

  • greeter = Mod.new

Instead, greeter needs to be an object of a class which has the module Mod

mixed in. Thus:
  • class Greeter
  • include 'mod'

  • def other_method
  • ....
  • end
  • end

  • greeter = Greeter.new
  • greeter.say_hello

It is also possible to mix in the module outside the class declaration, i.e., 'on the fly', using the method name 'extend' on the class instance.

Finally, it is worth noting that Modules, like Variables, can be constants, i.e., they can be used to mix in constant features, such as a group of variables, or a group of mathematical functions, or the like. From here, it's not far to the concept of passed 'blocks' and 'procs'. But I'll save that for another post.
